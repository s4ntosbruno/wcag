O que é acessibilidade?

Acessibilidade é a prática de garantir que produtos, serviços e ambiente sejam projetados e desenvolvidos para serem utilizados por todas as pessoas, independentemente de suas habilidades, limitações ou características individuais. Isso envolve a criação de condições que permitam o acesso e a participação de todos, incluindo pessoas com deficiências físicas, sensoriais, cognitivas, intelectuais ou outras diversas condições.

Na área digital, a acessibilidade se concentra em tornar site, aplicativos e conteúdos online acessíveis para pessoas com diferentes necessidades. isso inclui a implementação de recursos como descrições de imagens para pessoas cegas, ou com baixa visão, compatibilidade com leitores de tela, garantia de contraste para facilitar a leitura, navegação via teclado para quem não pode usar um mouse, entre outras práticas.

Dar acessibilidade é lei em alguns países, o que pode abrir mercados siginificativos.

Nome Acessível

É a atribuição de um nome descritivo e significativo a um elemento da interface, como um link, botão, imagem ou qualquer outro componente interativo, de modo que usuários que utilizam tecnologias assistivas, como leitores de tela, possam compreender e interagir facilmente com esse elemento.

  • Maneiras de adicionar nome acessível:
    • com conteúdo filho.
    • com um atributo string via aria-label.
    • referenciando o conteúdo com aria-labelledby.
    • adicionando o elemento label nos controles de formulário.
    • adicionando o elemento legenda nos fildsets.
    • adicionando legendas nas imagens e tabelas.
    • nomes alternativos de títulos e placeholders.

Diretrizes de Acessibilidade para Conteúdo da Web

WCAG são diretizes de acessibilidade para conteúdo na web, foram desenvolvidas em cooperação com o indivíduos e empresas do mundo inteiro, com o Objetivo de fornecer um único padrão compartilhado para acessibilidade de conteúdo da web que atenda às necessidades de indivíduos, organizações e governos internacionalmente.

o guia fornece diversas camadas de informação, são elas:

  • Princípios - fornecem a base para a acessibilidade na Web

  • Diretrizes - mostram os Objetivos básicos que os desenvolvedores devem trabalhar.

  • Critérios de Sucesso - Para cada diretriz, critérios são fornecidos para que o wcag seja utilizado onde requisitos e testes são necesários, como design e desenvolvimento.

  • Técnicas Suficientes e Auxiliares - Para cada um dos critérios de sucesso, exitem várias técnicas documentadas para resolver os problemas.

Princípios

  1. Perceptível:

    • O conteúdo e suas informações devem ser apresentados de maneira que os usuários possam percebê-los. Isso inclui garantir que todos os elementos sejam percebíveis por diferentes sentidos, como visão, audição ou tato.

  2. Operável:

    • Os componentes de interface e navegação devem ser operáveis pelos usuários. Isso implica que os usuários devem ser capazes de interagir e navegar pelo conteúdo, utilizando uma variedade de dispositivos de entrada, como teclado, mouse ou até mesmo por voz.

  3. Compreensível:

    • O conteúdo e a interface devem ser compreensíveis para que os usuários possam entender e utilizar as informações facilmente. Isso inclui tornar o texto claro e legível, bem como garantir a consistência e previsibilidade na interface do usuário.

  4. Robusto:

    • O conteúdo deve ser robusto o suficiente para ser interpretado por uma ampla variedade de tecnologias, incluindo tecnologias assistivas. Isso significa que ele deve ser compatível e acessível através de diferentes navegadores, dispositivos e tecnologias assistivas.

Diretrizes

  1. Perceptível:

    • 1.1 Alternativas de Texto: Oferecer alternativas textuais para elementos não textuais como imagens, vídeos e áudio.
    • 1.2 Mídia Baseada em Tempo: Fornecer alternativas para conteúdo de mídia baseada em tempo, como legendas para vídeos.
    • 1.3 *Adaptável: Criar conteúdo que possa ser apresentado de diferentes maneiras sem perder informações ou estrutura.
    • 1.4 Distinguível: Garantir que o conteúdo seja apresentado de maneira que seja facilmente distinguível por usuários com deficiências visuais, como contraste de cores suficiente.

  2. Operável:

    • 2.1 Acessível pelo Teclado: Tornar todo o conteúdo navegável e operável via teclado.
    • 2.2 Tempo Suficiente: Dar aos usuários tempo suficiente para ler e usar o conteúdo.
    • 2.3 Convulsões e Reações Físicas: Evitar conteúdo que possa desencadear convulsões ou reações físicas inesperadas.
    • 2.4 Navegável: Facilitar a navegação e localização do conteúdo.
    • 2.5 *Modalidades de Input: Tornar as funcionalidades acessíveis por meio de uma variedade de dispositivos de input.

  3. Compreensível:

    • 3.1 *Legível: Fazer com que o conteúdo textual seja legível e compreensível.
    • 3.2 Previsível: Fazer com que o conteúdo funcione de maneira previsível e consistente.
    • 3.3 Ajuda no Input de Dados: Auxiliar os usuários a evitar e corrigir erros.

  4. Robusto:

    • 4.1 *Compatível: Maximizar a compatibilidade com diferentes tecnologias e navegadores.
